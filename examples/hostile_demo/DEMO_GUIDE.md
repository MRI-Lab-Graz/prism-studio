# PRISM Hostile Demo

This project was generated by `prism_tools dataset build-hostile-demo` to deliberately exercise edge cases across sociodemographics, biometrics, environmental/MRI data, and subject/session identifiers. Each case below names the injected hostile value and the expected pipeline behavior.

## Sociodemographics

| Case ID | Location | Description | Expected Outcome |
|---|---|---|---|
| `socio_age_out_of_range` | `—` | Ages outside the documented 18-100 range (-5 and 999). | Converter does not crash; out-of-range values pass through for downstream BIDS-validator range checks to flag. |
| `socio_sex_menstrual_conflict` | `—` | sex='M' with menstrual_cycle_regularity populated (conditional field should not apply to this sex value). | Value is preserved as-is (converter does not currently enforce conditional-display logic); flags a gap for the BIDS validator or UI to catch, not the converter. |
| `socio_unicode_freetext` | `—` | Unicode, emoji, and an RTL-override control character in free text fields. | Round-trips through TSV without mangling or raising. |
| `socio_empty_participant_id` | `—` | Empty and whitespace-only participant id values. | Rows are dropped with a 'Dropped N rows without valid participant_id' message. |
| `socio_duplicate_id_case` | `—` | sub-01 and SUB-01: the prefix is case-normalized to lowercase by ParticipantsConverter._normalize_participant_id, so these resolve to the identical participant_id 'sub-01'. | Rows are collapsed into a single participants.tsv row via _collapse_to_bids_participants_table; conflicting non-empty values across the two rows are reported in conflicting_columns. |
| `socio_distinct_case_in_label` | `—` | sub-Ab vs sub-ab: case difference inside the label itself (not the 'sub-' prefix), which is NOT normalized away. | Two distinct participant_id rows are kept (sub-Ab, sub-ab) — no collapsing. |
| `socio_whitespace_only_id` | `—` | Participant id made only of tabs/newlines. | Dropped via the same path as empty ids. |
| `socio_extremely_long_id` | `—` | A 300-character participant id (path-length stress for downstream sub-<id>/ses-<id>/... directory creation). | Converter accepts the long label without crashing; downstream filesystem operations may hit OS path-length limits, which is the behavior under test at the pipeline stage, not here. |
| `socio_non_iso_country` | `—` | country_of_birth values that are not valid ISO 3166-1 alpha-2 codes. | Values pass through unvalidated at the converter layer; non-ISO codes remain a BIDS-validator/UI concern. |
| `socio_conflicting_conditional` | `—` | medication_current='no' but medication_details populated. | Contradiction passes through unflagged at the converter layer; no crash. |
| `socio_nan_inf_numeric` | `—` | NaN/inf/literal-string 'NaN' in height/weight/bmi. | Numeric coercion downstream does not crash; inf/NaN do not silently become a plausible-looking number. |
| `socio_csv_injection` | `—` | Spreadsheet-formula-like strings (=1+1, =cmd|'/c calc'!A1, @SUM(1+1), +1+1) in free-text fields. | Values are preserved literally as text by pandas/TSV writers; verify any xlsx export path does not auto-evaluate or strip the leading character. |
| `socio_education_years_out_of_range` | `—` | education_years outside the documented 0-30 range. | Out-of-range values pass through; no crash. |
| `socio_group_field_embedded_tab` | `—` | A literal tab character embedded inside the 'group' field, destined for TSV output. | pandas' TSV writer quotes/escapes the value so the TSV structure is not corrupted (column count stays correct). |

## Biometrics

| Case ID | Location | Description | Expected Outcome |
|---|---|---|---|
| `bio_duplicate_item_id_across_groups` | `—` | item_id 'resting_hr' appears under both 'fitness' and 'vitals' groups in the codebook. | Group resolution is ambiguous by design here; downstream code must pick one group deterministically or report the ambiguity, not crash or silently drop the item. |
| `bio_wide_long_ambiguous` | `—` | A row carries both wide per-item columns and a long-format item_id/value pair simultaneously. | Fuzzy column detection must pick one interpretation consistently rather than double-counting the row. |
| `bio_out_of_codebook_range` | `—` | resting_hr=500 and grip_strength_left=-10, both outside the codebook's documented min/max. | Values pass through to the biometrics TSV; range enforcement is a validator/UI concern, not a converter crash. |
| `bio_session_label_variants` | `—` | Sessions '1', '01', and 'pre' for different participants in the same file. | Each session label is preserved verbatim — never coerced or treated as equal to another (CLAUDE.md session-label rule). |
| `bio_unicode_item_values` | `—` | A numeric biometrics field unexpectedly contains free text ('N/A - refused'). | Pipeline does not crash on the non-numeric value in a numeric column; it is preserved or coerced to NaN, not silently treated as 0. |
| `bio_instance_non_numeric` | `—` | instance column contains a non-numeric value ('trial-A'). | Non-numeric instance values are coerced to NaN and the row is dropped from instance-pivoted output, not crashed on. |
| `bio_nan_value_cells` | `—` | An empty/None value cell for an item_id. | Serialized as the BIDS 'n/a' sentinel, not the literal string 'nan'. |
| `bio_missing_participant_id_column_variants` | `—` | Participant id column padded with noise text instead of a recognizable header name. | Column-detection fuzzy matching either recovers the column or fails with a clear, non-crashing error message. |

## Environment / MRI

| Case ID | Location | Description | Expected Outcome |
|---|---|---|---|
| `env_missing_acquisition_datetime` | `sub-env01/ses-01/anat/sub-env01_ses-01_T1w.json` | Sidecar has no AcquisitionDateTime and no Date/Time or StudyDate/Time fallback fields. | parse_sidecar_timestamp returns None cleanly; row is skipped, no exception. |
| `env_malformed_acquisition_datetime` | `sub-env01/ses-01/func/sub-env01_ses-01_task-rest_bold.json` | AcquisitionDateTime with an invalid month/day/hour (2026-13-45T99:99:99). | Timestamp parsing fails gracefully (returns None), not an unhandled ValueError. |
| `env_dst_crossing` | `sub-env02/ses-01/func/sub-env02_ses-01_task-dst_bold.json` | Timestamp at 2026-03-29T02:30:00, inside the Europe/Vienna spring-forward DST transition window. | Parses to a concrete datetime without raising on the ambiguous/nonexistent local time. |
| `env_leap_day_valid` | `sub-env03/ses-01/anat/sub-env03_ses-01_T1w.json` | AcquisitionDate=2024-02-29 (a real leap day). | Parses successfully to 2024-02-29T10:00:00. |
| `env_leap_day_invalid` | `sub-env03/ses-02/anat/sub-env03_ses-02_T1w.json` | AcquisitionDate=2026-02-29 — 2026 is not a leap year, so this date does not exist. | Parsing fails gracefully (returns None), not an unhandled exception. |
| `env_missing_institution_address` | `sub-env04/ses-01/anat/sub-env04_ses-01_T1w.json` | InstitutionName present, InstitutionAddress absent. | extract_sidecar_location falls back to InstitutionName without raising. |
| `env_conflicting_study_vs_acquisition_date` | `sub-env05/ses-01/anat/sub-env05_ses-01_T1w.json` | StudyDate (2026-02-20) differs from AcquisitionDate (2026-02-26) by 6 days in the same sidecar. | AcquisitionDate/Time takes precedence over StudyDate/Time; the two are not averaged or merged. |
| `env_subsecond_fraction_single_digit` | `sub-env06/ses-01/anat/sub-env06_ses-01_T1w.json` | Seconds field is a single-digit fractional value ('...T14:30:2.5') instead of zero-padded ('02.5'). | Seconds are zero-padded before parsing; no ValueError. |
| `env_timezone_naive_vs_aware_mix` | `sub-env07/ses-01/anat/sub-env07_ses-01_T1w.json` | One sidecar uses a 'Z'-suffixed UTC timestamp, another (same subject, different session) uses a naive local timestamp. | Both parse independently without raising; they are not silently treated as the same instant. |
| `env_weather_extremes` | `code/rawdata/hostile_environment_weather_extremes.csv` | Synthetic weather rows with values outside any real-world record (temp_c=-89.2, aqi=99999, pm25_ug_m3 empty/NaN) — written directly to avoid live Open-Meteo API calls in tests. | Downstream binning (hour_to_bin/season_code/etc.) does not raise on extreme or missing numeric values. |

## Subject / Session IDs + DataLad

| Case ID | Location | Description | Expected Outcome |
|---|---|---|---|
| `subj_case_collision_no_merge` | `sub-01/ses-01/func/sub-01_ses-01_task-rest_bold.json` | A subject directory 'sub-01' that a rewrite plan targets via a pure case-only rename (e.g. from 'SUB-01'). | SubjectCodeRewriter must not treat a pure case-only rename as colliding with itself, but must still detect genuine collisions when a distinct, unrelated subject maps to the same target (see tests/test_hostile_demo_pipeline.py). |
| `subj_many_to_one_merge_candidate` | `sub-AAA/ses-01/func/sub-AAA_ses-01_task-rest_bold.json` | A subject directory 'sub-AAA' used as one of three sources (alongside hypothetical 'sub-aaa', 'sub-Aaa') that would collapse to the same case-insensitive target under a rewrite rule. Real co-existing case-variant directories cannot be staged on a case-insensitive filesystem, so the merge path is exercised via explicit_mapping in the pipeline tests. | Rewriting with allow_many_to_one=False reports conflicts; with allow_many_to_one=True, directories are merged without data loss. |
| `subj_non_ascii_id` | `sub-Müller/ses-01/func/sub-Müller_ses-01_task-rest_bold.json` | Subject directory 'sub-Müller' (non-ASCII character in the label). | The rewriter's subject-token regex (sub-[A-Za-z0-9]+) does not match non-ASCII labels, so this directory is silently skipped by rewrite operations rather than crashing — a known gap to be aware of, not a pipeline failure. |
| `sess_no_coercion_1_vs_01_vs_pre` | `—` | Same subject (sub-sess01) with three sessions literally named ses-1, ses-01, and ses-pre. | All three remain distinct, exact-string session labels throughout every pipeline stage (participants, biometrics, environment, recipes, export) — never coerced or treated as equal, per the repository's session-label rule. |
| `subj_extremely_long_subject_id` | `—` | A 204-character subject directory name. | Filesystem path creation succeeds; downstream pipeline code does not silently truncate the id. |

## Regenerating

```
python prism_tools.py dataset build-hostile-demo --output /Users/karl/work/github/prism-studio/examples/hostile_demo --guide
```

Generation is deterministic for a fixed `--seed`, so regenerating with the same seed should reproduce byte-identical text/tabular output (binary formats such as `.xlsx` may still differ).
